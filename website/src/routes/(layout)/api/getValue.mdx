# getValue

Returns the value of the specified field.

```ts
const value = getValue(form, name, options);
```

## Parameters

- `form` <Property {...properties[getFramework()]?.form} />
- `name` <Property type="string" />
- `options` <Property {...properties.options} />
  - `shouldActive` <Property {...properties.shouldActive} />
  - `shouldTouched` <Property {...properties.shouldTouched} />
  - `shouldDirty` <Property {...properties.shouldDirty} />
  - `shouldValid` <Property {...properties.shouldValid} />
  - `typeValidated` <Property {...properties.typeValidated} />

### Explanation

By default `undefined` is returned if the field is inactive. To change this behavior you can set `shouldActive` to `false`.

If you want `undefined` to be returned when the field is not touched, not dirty and/or not valid, you can set `shouldTouched`, `shouldDirty` and/or `shouldValid` to `true`.

`shouldActive`, `shouldTouched`, `shouldDirty` and `shouldValid` act like a kind of filter and can be combined with each other.

<Solid block>

`typeValidated` is for TypeScript users only. By default, the value is returned as maybe `undefined` because we have no way of knowing whether you actually used the specified field and whether your user's input is valid. If you are sure that the returned value matches your type definition of the form values, for example because they have been validated before, you can tell us to cast it to your type definition by setting `typeValidated` to `true`.

</Solid>

## Return

- `value` <Property {...properties.value} />
