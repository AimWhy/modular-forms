# FormState

Type that defines the state of a form.

## Definition

- `FormState` <Property type="object" />
  - `element` <Property {...properties.element} />
  - `submitCount` <Property type="number" />
  - `submitting` <Property type="boolean" />
  - `submitted` <Property type="boolean" />
  - `validating` <Property type="boolean" />
  - `touched` <Property type="boolean" />
  - `dirty` <Property type="boolean" />
  - `invalid` <Property type="boolean" />
  - `response` <Property {...properties.response} />

> The object also contains a key named `internal` which provides API's that we need internally in the library. You should not access it. Instead, use the form, field and array methods we provide you.

### Explanation

The state properties of the form are reactive and can be used, for example, in the JSX area or Solid's `createEffect` primitive.

If you use the <A href="/api/Form">`Form`</A> component or the <A href="/api/handleSubmit">`handleSubmit`</A> method to submit your form, errors that are thrown are made accessible via the `response` object. In conjunction with the <A href="/api/setResponse">`setResponse`</A> method, you can also use `response` to display custom feedback to your user.
